Static Sites with Org-page

First Post

Why?

For quite some time now I have wanted to create a website were I could post the occasional observation about topics that were either tangential to my science, more fanciful and speculative, or just "off-topic". For example, I am a big emacs fan, but it doesn't seem smart to clutter posts about what the lab group has been up to by posting on https://brittlab.uwaterloo.ca. I decided therefore to create a more experimental website where I could post these digressions. It is intended to be an opportunity to experiment more with blogging as a way of sharing ideas, and also to experiment with site creation.

How

As many who have marched down this trail will confirm it is easy to get waylaid, and find yourself in the high brush completely lost. There are a lot of options for static site generation, and for the Lab Groups site we are using Jekyll, which frankly works very well. This seemed a good choice for the Lab site, because that website has a secondary purpose of education. I wanted the students to be able to participate in its updating, and configuration, and I thought that the popularity of Jekyll would make it more portable and useful for them in the long term.

But this site was to be for me, and I have been working very hard to consolidate the number of systems with which I work and the amount of special purpose mappings (be they hotkeys or program logic) that I had to learned. I have been gradually pulling more and more of my workspace into Emacs, and it seemed therefore to find a way to try and do things with Emacs. One of the tools I have been using a lot lately is org-mode, and so I wanted if possible to make the website with org-mode. Or at the very least to use org-mode for the markup dialect. There are a few choices on this route.

  1. Use org-mode markup to write your posts and then pandoc to convert to whatever dialect of markup/down you need.
  2. Use org-mode completely by invoking org-publish.
  3. Use a tool built on top of org-mode, such as org-page.

I did not do enough experimenting to justify a strong opinion. Choice number 1 is fine, and is probably the simplest. If your goal is to just to get up and running then use something with a large userbase where it will be easy to find support and help (e.g. Jekyll). But I also wanted to do a bit more learning about site creation so I felt left with 2 and 3. I think I will try org-publish, but for now it seemed to require more effort than I felt I had time for. I am interested in having support for comments, analytics, RSS feeds, and so I chose something where those were built in, thus org-page.

The creation experience with org-page was very smooth. I just downloaded the package from MELPA, and defined the following in my init file. 

;;org-page configuration
(require 'org-page)
(setq op/repository-directory "/home/britt/gitRepos/brittGitSite")
(setq op/site-domain "https://brittAnderson.github.io")
(setq op/personal-discus-shortname "brittGitSite")
(setq op/site-main-title "Britt Anderson")
(setq op/site-sub-title "Naively regarding the external world")
(setq op/personal-github-link "https://github.com/brittAnderson")

Then I initiated everything with the built in helper function op/new-repository. And if you are reading this, then so-far so good.
